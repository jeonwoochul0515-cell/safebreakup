## 1. 법적 증거 수집 규정

### 1-1. 디지털 증거의 법적 효력

개인이 수집한 디지털 증거(스마트폰 사진, 영상, 문자 캡처, 녹음)가 법적 증거로 인정되려면 다음 요건 충족 필수:

| 요건 | 설명 |
|------|------|
| **원본성** | 원본 파일이거나, 원본과 동일한 사본임이 입증되어야 함 |
| **무결성** | 수집 시점부터 법정 제출 시까지 변경/편집되지 않았음이 담보되어야 함 |
| **동일성** | 원본 저장매체의 내용과 제출된 증거물의 내용이 동일해야 함 |
| **신뢰성** | 수집 과정에서 사용된 장비와 방법의 정확성이 보장되어야 함 |
| **적법성** | 위법한 방법으로 수집되지 않아야 함 (위법수집증거배제법칙) |

### 1-2. 형사소송법상 디지털 증거 요건

**형사소송법 제313조 (진술서등):**
- **제1항**: 진술서는 작성자의 공판 진술에 의해 성립의 진정함이 증명된 때 증거 사용 가능
- **제2항 (2016년 개정)**: 진술서 작성자가 성립의 진정을 부인하더라도, **디지털포렌식 등 객관적 방법**으로 진정성립 증명 가능

### 1-3. 대법원 핵심 판례

| 판례 | 핵심 법리 |
|------|-----------|
| **대법원 2007도7257** | 디지털 저장매체에서 출력한 문건의 증거능력에는 원본과의 **동일성** 인정 필요. 압수 시부터 출력 시까지 **변경 없음** 담보 필수 |
| **대법원 2017도13263** | **해시값** 등으로 무결성·동일성 검증. 입증책임은 검사에게 있으며, **합리적 의심 배제** 수준까지 요구 |
| **대법원 2015도1900** | 대화 당사자 일방의 녹음은 "타인간의 대화"에 해당하지 않으므로 **합법** |

### 1-4. 녹음의 적법성

| 유형 | 적법성 | 근거 |
|------|--------|------|
| 대화 당사자 일방이 녹음 | **합법** | 통신비밀보호법 제3조 — "타인간의 대화"에 해당하지 않음 |
| 제3자가 동의 없이 녹음 | **불법** | 통신비밀보호법 제3조 제1항, 제14조 제1항 위반 |
| 제3자가 일방의 동의만 받고 녹음 | **불법** | 대법원 2015도1900 — 상대방 동의 없으면 감청에 해당 |

위반 시: **1년 이상 10년 이하의 징역 + 5년 이하의 자격정지** (제16조 제1항 제2호)
불법 녹음은 **재판에서 증거 사용 불가** (통신비밀보호법 제4조)

### 1-5. 앱 생성 PDF 보고서의 법적 효력

**전자문서 및 전자거래 기본법 제4조 제1항:**
> "전자문서는 전자적 형태로 되어있다는 이유로 법적 효력이 부인되지 아니한다"

**현실적 판단:**

| 항목 | 결론 |
|------|------|
| PDF 자체의 법적 효력 | 전자문서로서 효력은 인정되나, **독립적 증거능력은 제한적** |
| 경찰 제출 가능 여부 | **참고자료/보조자료**로 충분히 활용 가능 |
| 증거능력 확보 조건 | 원본 데이터(사진, 녹음, 캡처 등)와 함께 제출하면 증거력 강화 |
| 해시값의 역할 | **무결성 입증의 유력한 수단**이나, 그것만으로 증거능력 자동 인정은 아님 |

**권고:** 공인 타임스탬프 서비스(TSA) 활용으로 신뢰성 보강

### 1-6. 스토킹 사건에서 효과적인 증거 (우선순위)

| 순위 | 증거 유형 | 설명 |
|------|-----------|------|
| 1 | **시간순 정리된 문자/SNS 메시지** | "지속적, 반복적" 행위 입증 핵심 |
| 2 | **CCTV 영상** | 따라다니기, 주거/직장 부근 배회 등 객관적 증거 |
| 3 | **녹음 파일** | 협박, 욕설, 위협 내용 (당사자 녹음만 합법) |
| 4 | **사진/영상** | 스토킹 현장 촬영, 택배/물건 배달 증거 |
| 5 | **진단서** | 스토킹으로 인한 상해/정신적 피해 |
| 6 | **목격자 진술** | 제3자 목격 진술서 |
| 7 | **112 신고 이력** | 반복성 입증에 도움 |

> **핵심:** 스토킹범죄의 성립 요건 "지속적 또는 반복적"을 입증하는 **시간순 누적 기록**이 가장 결정적

## 2. 개인정보보호 규정

### 2-1. 개인정보보호법 적용 여부

앱 운영자(법률사무소 청송)는 **개인정보처리자**(법 제2조 제5호)에 해당.
**로컬 저장만 하더라도** 앱 운영자가 수집 로직을 설계한 이상 개인정보보호법 **전면 적용**.

### 2-2. 수집/이용 동의 요건 (제15조)

| 법적 근거 | 내용 | 적용 |
|-----------|------|------|
| 제15조 제1항 제1호 | 정보주체의 **동의** | 피해자(사용자) 개인정보 — 회원가입 시 동의 |
| 제15조 제1항 제3호 | **법률에 특별한 규정** | 스토킹처벌법, 성폭력처벌법 등 증거 보전 목적 |
| 제15조 제1항 제6호 | **정당한 이익** | 피해자 보호 공익적 목적 주장 가능 |

동의 시 필수 고지: 수집 목적, 수집 항목, 보유 기간, 동의 거부권 및 불이익

### 2-3. 민감정보 처리 제한 (제23조) — ⚠️ 중요

| 데이터 | 민감정보 해당 | 근거 |
|--------|:------------:|------|
| 폭행 피해 사진(상해) | **해당** | 건강정보 |
| 성폭력 피해 기록 | **해당** | 성생활 정보 |
| 정신과 치료 기록 | **해당** | 건강정보 |
| 스토킹 피해 일지 | **가능성 높음** | 사생활 현저히 침해 우려 |

**필수 조치:** 민감정보 수집 시 **일반 동의와 별도의 동의 화면** 구현 필수

### 2-4. 제3자(가해자) 개인정보 문제

가해자의 이름, 전화번호, 사진이 포함된 증거 보관 = 가해자 개인정보 **동의 없이 처리**

**동의 없이 저장할 수 있는 법적 근거:**
- **제15조 제1항 제3호**: 스토킹처벌법, 형사소송법 제221조(증거 보전) 등 법률의 특별한 규정
- **제15조 제1항 제6호**: 피해자 보호라는 정당한 이익이 가해자의 개인정보자기결정권보다 우선

**⚠️ 제58조(적용 제외)는 민간 앱에 적용 불가.** 범죄 수사 목적의 제외는 수사기관에만 해당.

**리스크 완화 방안:**
- 증거 데이터는 **로컬 저장 기본** → 앱 운영자가 가해자 정보에 직접 접근 불가 구조
- 서버 업로드 시 **E2E 암호화** 적용 → 운영자도 내용 열람 불가
- 개인정보처리방침에 "제3자 정보 포함 증거의 보관 목적 및 법적 근거" 명시

### 2-5. PDF 보고서 생성/공유 시 개인정보

- PDF에 가해자 정보 포함 시 **제3자 제공(제17조)** 해당
- 경찰 제출 시 적법 근거: 형사소송법 제234조(고소권), 제237조(고발)
- **피해자가 다운로드 후 직접 제출하는 구조 권장** (앱이 직접 공유하지 않음)
- 불필요한 정보 **마스킹 처리** 옵션 제공

### 2-6. 데이터 저장 방식별 규정

| 구분 | 로컬 저장 (기기 내) | 클라우드 저장 (서버) |
|------|:------------------:|:------------------:|
| 개인정보보호법 적용 | **적용** | **적용** + 추가 의무 |
| 위탁 계약 (제26조) | 불필요 | 수탁자와 문서 계약 필수 |
| 국외 이전 (제28조의8) | 해당 없음 | 해외 서버 시 동의 필요 |

### 2-7. 암호화 의무 (안전성 확보조치 기준 고시)

| 대상 정보 | 암호화 의무 | 알고리즘 |
|-----------|:----------:|----------|
| 비밀번호 | 일방향 암호화 필수 | SHA-256 이상 |
| 고유식별정보 | 암호화 필수 | AES-128 이상 |
| 민감정보 | 암호화 필수 | AES-128 이상 |
| 네트워크 전송 | 보안서버 필수 | TLS 1.2 이상 |

> **🚨 현재 앱 상태: 암호화 미구현 (Base64 시뮬레이션).** 출시 전 반드시 AES-256-GCM 실제 구현 필수.
> 위반 시: **2년 이하 징역 또는 2천만원 이하 벌금** (법 제73조)

### 2-8. 앱 운영자 의무 요약

| 항목 | 의무 수준 | 내용 |
|------|:---------:|------|
| 개인정보처리방침 (제30조) | **필수** | 처리 목적, 보유 기간, 제3자 제공, 파기 절차, 보호책임자 등 기재 |
| 보호책임자 지정 (제31조) | **필수** | 대표변호사 또는 임원 지정 후 공개 |
| 파기 의무 (제21조) | **필수** | 목적 달성 시 5일 이내 파기, 복구 불가능 조치 |
| 개인정보 영향평가 (제33조) | **노력 의무** | 민간은 법적 의무 아니나 자발적 실시 강력 권장 |
| ISMS-P 인증 | **초기엔 불필요** | 매출 100억 이상 또는 이용자 100만 이상 시 의무 |

## 3. 시장성 분석

### 3-1. 국내외 유사 서비스

| 서비스 | 국가 | 핵심 기능 | 가격 | 상태 |
|--------|------|-----------|------|------|
| **DocuSAFE** (NNEDV) | 미국 | 학대 증거 기록/저장/공유 | 무료 | **2023.10 종료** |
| **EviSafe** | 네덜란드 | AI 기반 폭력 인식 녹음, 타임스탬프 | 무료(기본) | 파일럿 운영 중 |
| **Bright Sky** (Vodafone) | 영국 | My Journal(증거 기록), 안전 설문 | **완전 무료** | 100만+ 사용자, 13개국 |
| **bSafe** | 노르웨이 | SOS, Follow Me, 녹음, 스트리밍 | 무료+구독(연$20) | 125개국 |

**한국 시장: "증거 수집 + 법률 AI + 피해자 보호" 결합 앱 전무. 명확한 블루오션.**

### 3-2. 시장 수요 — 급증하는 피해 통계

**스토킹/교제폭력 신고 추이:**

| 연도 | 스토킹 신고 | 스토킹+교제폭력 합계 |
|------|:-----------:|:-------------------:|
| 2022년 | 29,565건 | 100,355건 |
| 2023년 | ~30,000건 | 108,974건 |
| 2024년 | 31,947건 | **120,341건** |
| 2025년 1~7월 | — | **79,131건** (전년 대비 +20%) |

**기타 통계:**
- 교제폭력 신고: 2020년 49,225건 → 2023년 **77,150건** (+57%)
- 데이트 폭력범: 2024년 **9,832명** 검거 (매일 37명꼴)
- 가해자 연령: **20대 36.8%**, 30대 25.6%
- 여성긴급전화 1366: 2024년 상담 **29만 건**

### 3-3. 타겟 사용자 규모 추정

| 구분 | 규모 |

| 스토킹+교제폭력 연간 신고자 | ~12만 명 |
| 미신고 포함 추정 피해자 (신고율 10~20%) | **60~120만 명** |

### 3-4. 리걸테크 시장

- 글로벌: 2027년 **356억 달러(47조 원)** → 2032년 **약 90조 원**
- 국내 법률시장: 약 **8조 1,800억 원**
- 로톡 매출: 2024년 약 **100억 원** (전년 대비 72% 성장)

### 3-5. 경쟁 우위

| 기존 방법 | 문제점 | 안전이별의 해결 |
|-----------|--------|----------------|
| 수기 기록 | 시간순 정리 어려움, 증거력 약함 | 자동 타임스탬프, 체계적 정리 |
| 변호사 의뢰 | 비용 부담 (상담비 수만원~) | AI 챗봇으로 즉시 법률 가이드 |
| 경찰 신고 | 증거 부족으로 불기소 다수 (구속률 3.3%) | PDF 증거 보고서로 신고 효과 극대화 |
| 캡처/녹음 | 산발적 저장, 분실·삭제 위험 | 암호화 보관, 위장 아이콘(계산기) |

## 4. 가격정책 검토

### 4-1. 윤리적 문제 — 피해자에게 비용을 받는 것

**위험 요소:**
- "피해자에게 돈을 받는다"는 **사회적 비판 가능성 높음**
- 해외 유사 서비스(Bright Sky, DocuSAFE)는 **모두 무료**
- 한국에 무료 법률구조(대한법률구조공단, 성폭력위기센터)가 이미 존재

**결론: 핵심 안전 기능은 반드시 무료. 수익은 부가 서비스 + B2B + 정부 지원으로.**

## 핵심 결론

### 법적 타당성: ✅ 가능하나 조건부
- 앱 생성 PDF는 **참고자료/보조자료**로 법적 활용 가능
- 독립적 증거능력은 제한적이므로, **원본 데이터와 함께 제출** 필요
- 해시값 + 타임스탬프로 무결성 입증 보강 가능

### 개인정보: ⚠️ 민감정보 대량 처리로 높은 규제 리스크
- 현재 암호화 미구현 상태는 **법 위반** — 출시 전 반드시 해결
- 가해자 정보 보관은 "정당한 이익" 또는 "법률의 특별한 규정"으로 근거 확보 가능
- 로컬 저장 + E2E 암호화 구조가 리스크 최소화에 최적

### 시장성: ✅ 명확한 블루오션
- 한국에 동일 서비스 전무, 해외에서도 DocuSAFE 종료 후 공백
- 스토킹+교제폭력 신고 연 12만 건, 매년 20% 증가
- 잠재 타겟 60~120만 명

### 가격정책: 핵심 기능 무료 + 복합 수익 모델
- 피해자 대상 안전 기능은 **무료가 업계 표준**
- 수익: **정부 보조금 + B2B(변호사 SaaS) + 프리미엄 구독 + CSR 후원**
- 법률사무소 직접 운영 구조가 핵심 차별점 (변협 규제 회피 + 변호사 직접 연결)

## 6. 대법원 핵심 판례 — 경미한 행위도 누적하면 스토킹

### 대법원 2023도6411 (2023.9.27. 판결, 형사2부, 주심 천대엽)

> **"경미한 행위도 누적·반복해 공포심 느끼면 스토킹 범죄"**

#### 사실관계
- A씨는 이혼한 전 배우자 B씨의 집에 약 1개월간 **6회 방문**
- 문을 열어달라고 소리치는 등 접근
- B씨는 과거 A씨에게 성폭력 피해를 당해 **접근금지 명령**을 신청한 전력
- 1심: 징역 1년 + 치료프로그램 40시간
- 2심: 징역 10개월 + 치료프로그램 40시간
- **대법원: 원심 확정**

#### 핵심 판시사항 (3가지 최초 판시)

**1. 스토킹범죄는 "위험범"이다**
> 스토킹범죄는 행위자의 행위를 매개로 상대방에게 불안감 또는 공포심을 일으킴으로써 자유로운 의사결정의 자유 및 생활형성의 자유와 평온이 침해되는 것을 막는 **위험범**이다.

**2. 현실적 공포심 불필요 — 객관적·일반적 기준**
> 객관적·일반적으로 볼 때 상대방으로 하여금 불안감 또는 공포심을 일으키기에 **충분한 정도**라고 평가될 수 있다면, **현실적으로 상대방이 불안감 내지 공포심을 갖게 되었는지 여부와 관계없이** 스토킹행위에 해당한다.

**3. 경미한 행위도 누적·포괄적 평가**
> 비교적 **경미한 수준의 개별 행위**라 하더라도 그러한 행위가 **반복되어 누적**될 경우 상대방이 느끼는 불안감 또는 공포심이 **비약적으로 증폭**될 가능성이 충분하다. 단기간에 수차례 반복된 행위는 **누적적·포괄적으로** 불안감 또는 공포심을 일으키기에 충분한 일련의 행위로 평가할 수 있다.

#### 판단 기준 (종합 고려 요소)
- 행위자와 상대방의 **관계·지위·성향**
- 행위에 이르게 된 **경위**
- 행위 **태양** (방식·정도)
- 행위자와 상대방의 **언동**
- 주변의 **상황** 등 행위 전후의 여러 사정

#### 안전이별 앱에 대한 시사점

| 시사점 | 적용 |
|--------|------|
| **경미한 행위도 증거로 수집 가치 있음** | "이 정도는 아닐 것 같아요"라는 피해자에게 → "경미해 보여도 누적되면 범죄입니다" 안내 |
| **현실적 공포심 입증 불필요** | 피해자가 "무섭진 않은데..."라고 해도 → 객관적으로 충분하면 스토킹 성립 |
| **누적 기록이 핵심** | 증거보관함의 **시간순 누적 기록**이 판례가 요구하는 "누적적·포괄적 평가"의 직접 증거 |
| **단기간 반복도 인정** | 1개월 6회 방문만으로 유죄 → 빈도가 낮아도 반복이면 충분 |

#### 사무장 대화에 반영할 멘트
> "혹시 '이 정도는 스토킹이 아닐 것 같다'고 생각하고 계신가요? 대법원은 2023년에 '경미한 행위라도 반복되면 스토킹 범죄'라고 명확히 판결했어요(2023도6411). 작은 것도 빠짐없이 기록해 두시는 게 중요합니다."

## 8. 행위중지 요청 서류 및 단계별 전략

### 8-1. 단계별 대응 전략

```
[1단계] 내용증명/경고장 발송 → 60~70% 여기서 해결
    ↓ (행위 계속 시)
[2단계] 112 신고 → 긴급응급조치 (100m 접근금지, 연락금지)
    ↓ (행위 계속 시)
[3단계] 고소 → 잠정조치 + 민사 가처분
    ↓ (유죄 판결 시)
[4단계] 형사 처벌 + 민사 손해배상
```

### 8-2. 스토킹처벌법상 보호조치

| 조치 | 발동 주체 | 내용 | 기간 | 위반 시 |
|------|-----------|------|------|---------|
| **응급조치** (제3조) | 경찰 (현장) | 제지, 분리, 서면경고 | 즉시 | — |
| **긴급응급조치** (제4조) | 경찰 (직권/요청) | 100m 접근금지, 연락금지 | 1개월 | **1년↓ 징역 / 1천만원↓** |
| **잠정조치** (제9조) | 검사→법원 | 접근금지, 연락금지, **전자발찌**, 유치 | 3개월 (최대 9개월) | **2년↓ 징역 / 2천만원↓** |

### 8-3. 변호사 명의 발송 서류

| 서류 | 핵심 역할 | 법적 효력 |
|------|-----------|-----------|
| **법률 경고장** | 행위 중단 요구 + 법적 조치 예고 | 직접 강제력 없으나, **"상대방의 의사에 반하여" 구성요건 입증의 결정적 증거** |
| **내용증명** (우편) | 경고장을 우체국이 공적 증명 | 발송 사실·날짜·내용을 3년간 보관 |
| **연락중지 요청서** | 일체의 연락 중단 요구 | 이후 연락 시 스토킹 성립 강화 |
| **접근금지 요청서** | 물리적 접근 금지 요구 | 이후 접근 시 스토킹 성립 강화 |

### 8-4. 내용증명이 핵심인 이유

> **내용증명 발송 후에도 연락/접근이 계속되면:**
> - "상대방의 의사에 반하여" 요건이 **자동 입증**
> - "거부 의사를 몰랐다"는 항변이 **차단**
> - 고소 시 유죄 판결 가능성 **현저히 높아짐**
> - 대법원 2023도6411 판례의 "누적적·포괄적 평가" 근거 강화

### 9-5. "보호 점수" 제안

증거 수집량 + 체크리스트 완료율 + 안전계획 이행도 + 법적 조치 진행도를 종합한 0~100 점수:

```
보호 점수 = (증거 수집 25%) + (체크리스트 25%) + (안전계획 25%) + (법적 조치 25%)

예: 증거 5건/10건(12.5) + 체크리스트 8/12(16.7) + 안전계획 미수립(0) + 경고장 발송(25) = 54점
```

+ "다음 행동 추천": "증거 2건 더 수집하면 보호 점수 +5"

### 9-6. 해외 사례 시사점

| 증거 유형 | 1차 처리 | 2차 AI 분석 | 건당 비용 |
|-----------|----------|------------|:---------:|

**월 비용 추정:**
- 100명 x 10건/월 = ~$5-15
- 1,000명 x 10건/월 = ~$50-150
- 10,000명 x 10건/월 = ~$500-1,500

### 10-3. 증거 유형별 AI 분석 범위

**문자/카톡 캡처:**
- OCR 텍스트 추출 → 협박/스토킹 키워드 감지 → 적용 법조 자동 매칭
- 정확도: OCR 90%+, 법적 분류 85%+
- 한계: 이모티콘/스티커 OCR 불가, 은어 해석 제한

**녹음 파일:**
- STT 텍스트 변환 → 화자 분리 → 위협 내용 분석 → 법적 분류
- 정확도: 한국어 STT 90%+ (조용한 환경)
- 한계: 사투리, 배경 소음 시 70-80%

**사진 (상해 증거):**
- Vision AI 설명 생성 → "멍으로 보이는 부위 확인" 수준
- 의학적 진단 불가 → "참고용, 진단서 대체 불가" 필수 고지

**PDF 문서:**
- 텍스트 추출 → 법적 조항 매칭 → 계약서/각서 분석
- 스캔 PDF는 OCR 의존 (85-90%)

### 10-4. 개인정보보호 — 하이브리드 방식 추천

| 방식 | 장점 | 단점 |
|------|------|------|
| 클라우드 AI | 높은 분석 품질, 빠른 구현 | 개인정보 외부 전송, 동의 필요 |
| 온디바이스 AI | 데이터 유출 없음 | 분석 품질 제한, 모델 100MB+ |
| **하이브리드 (추천)** | 1차 온디바이스 → 동의 시 클라우드 | 구현 복잡도 증가 |

**필수 동의:** "귀하의 증거 파일이 분석을 위해 외부 AI 서비스로 전송됩니다" 별도 동의
**Claude API:** 사용자 데이터를 모델 학습에 사용하지 않음 (개인정보 보호에 유리)

### 10-5. AI 분석 결과의 법적 지위

| 구분 | 법적 지위 |
|------|-----------|
| AI 분석 보고서 | 직접적 증거력 없음 (**보조 자료**) |
| AI 추출 텍스트 (OCR/STT) | 원본 증거의 가독성 보조 |
| AI 법적 분류 판단 | **참고 의견** (변호사/판사 대체 불가) |
| 원본 파일 + SHA-256 해시 | 무결성 입증 시 **증거력 인정 가능** |

대법원 '사법부 AI 활용 지침' (2025.2): AI 사용 사실 공개 의무, 신뢰성·정확성 검증 필요

### 10-6. 구현 아키텍처

```
[사용자] → 파일 선택/촬영/녹음
    ↓
[expo-image-picker / document-picker / audio]
    ↓
[expo-file-system] → 로컬 암호화 저장 (secure-evidence.ts)
    ↓
[SHA-256 해시 생성] → 무결성 보장
    ↓
[1차: 온디바이스] → 기본 OCR + 키워드 매칭 (오프라인 가능)
    ↓ (사용자 동의 시)
[2차: 클라우드 AI] → Claude Vision/Haiku + Whisper STT
    ↓
[분석 결과] → 법적 분류, 위험도, 관련 법조항, 다음 행동 추천
```

### 10-7. 해외 사례

## 11. 회원가입/로그인 + 결제 + 어드민 시스템

### 11-1. 인증 — Supabase Auth + Expo

**권장 인증 방식 (한국 사용자):**

| 순위 | 방식 | Supabase 지원 | Expo 호환 |
|:----:|------|:------------:|:---------:|
| 1 | **카카오 로그인** | 공식 | WebBrowser OAuth |
| 2 | **이메일/비밀번호** | 기본 | 네이티브 |
| 3 | **애플 로그인** | 공식 | expo-apple-authentication |
| 4 | 구글 로그인 | 공식 | expo-auth-session |

**SecureStore 2048바이트 제한 해결:** 암호화 키만 SecureStore, 세션은 암호화 후 AsyncStorage

### 11-2. 회원가입 법적 요건

필수 동의: 이용약관(필수) + 개인정보 수집/이용(필수) + 14세 이상(필수) + 마케팅(선택)
탈퇴 시: 즉시 삭제 + 결제 기록 5년 보존 + 증거 다운로드 안내

### 11-3. 결제 — 토스페이먼츠 확정

| 결제 유형 | 방식 | 결제수단 |
|-----------|------|---------|
| 구독 (월 9,900원) | 토스페이먼츠 정기결제 (빌링키) | 카드, 카카오페이, 토스페이 |
| 건별 결제 | 토스페이먼츠 일반결제 | 카드, 카카오페이, 토스페이, 네이버페이, 계좌이체 |

수수료: ~2.5-3.5% (앱스토어 30%보다 훨씬 저렴)

### 11-4. 피해자 보호 보안

- 앱 레벨 PIN/생체인증 (세션과 별개)
- 증거 삭제 72시간 유예
- 로그인 알림 + 원격 세션 무효화
- 패닉 버튼 (특정 PIN 시 데이터 숨김)
